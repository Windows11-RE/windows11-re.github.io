# 🚀 快速测试指南

## 测试富文本编辑器

请按以下顺序测试，找出问题所在：

### 1️⃣ 测试 Quill 基础功能（最简单）
**打开文件：** `test-quill.html`

**预期结果：**
- 看到一个带工具栏的编辑器
- 有示例文字"Quill 编辑器测试"
- 可以编辑文字
- 状态显示"✅ Quill 编辑器加载成功！"

**如果失败：** 说明 Quill 库无法加载，可能是网络问题。

---

### 2️⃣ 测试简化版编辑器
**打开文件：** `editor-simple.html`

**预期结果：**
- 看到完整的编辑器界面
- 有"欢迎使用 Quill 编辑器"的内容
- 可以点击按钮获取/设置/清空内容

**如果失败：** 说明 Quill 初始化有问题。

---

### 3️⃣ 运行诊断工具
**打开文件：** `editor-diagnostic.html`

**预期结果：**
- 环境检测：显示浏览器信息和在线状态
- CDN 测试：至少一个 CDN 显示"✅ 可用"
- Quill 库检测：显示"✅ Quill 库已加载"
- 点击"测试初始化"后，下方出现可编辑的编辑器

**如果失败：** 查看具体哪一项失败，按照提示处理。

---

### 4️⃣ 测试完整编辑器
**打开文件：** `editor.html`

**前提条件：**
- 必须先登录（打开 `login.html`，密码：admin123）
- 前面三个测试都通过

**预期结果：**
- 看到文章编辑页面
- "文章内容" 下方有一个富文本编辑器
- 编辑器有完整的工具栏（标题、粗体、颜色等）
- 可以输入和格式化文字

**如果失败：** 
1. 按 F12 打开控制台，查看错误信息
2. 查看是否有红色错误
3. 查看是否有"Quill 编辑器初始化成功"的日志

---

## 🔍 问题诊断

### 问题 A：所有测试页面都看不到编辑器

**原因：** CDN 无法访问

**解决方案：**
1. 检查网络连接
2. 尝试使用 VPN
3. 或者下载 Quill 到本地（见 `EDITOR_TROUBLESHOOTING.md`）

---

### 问题 B：测试页面能看到，但 editor.html 看不到

**原因：** 页面代码有冲突或初始化失败

**解决方案：**
1. 打开浏览器控制台（F12）
2. 查看 Console 标签的错误信息
3. 查找包含"Quill"或"editor"的错误
4. 根据错误信息排查

**常见错误：**
- `protectPage is not defined` → auth.js 未加载
- `Cannot read property 'innerHTML'` → 容器元素未找到
- `Quill is not defined` → Quill 库未加载

---

### 问题 C：编辑器显示但样式错乱

**原因：** CSS 文件未正确加载

**解决方案：**
1. 检查 `quill.snow.css` 是否加载
2. 在控制台 Network 标签查看资源加载状态
3. 尝试强制刷新（Ctrl+F5）

---

## 📋 控制台日志检查

打开 `editor.html` 后，控制台应该显示：

```
页面 DOM 加载完成
准备初始化 Quill 编辑器
开始初始化 Quill 编辑器...
Quill 编辑器初始化成功！ Quill {…}
加载文章数据到编辑器
```

如果看到这些日志，说明编辑器初始化成功。

---

## ✅ 成功标志

编辑器正常工作时，你应该看到：

```
┌─────────────────────────────────────────┐
│ [H] [B] [I] [U] [S] [色] [背景] ...    │ ← 工具栏
├─────────────────────────────────────────┤
│                                         │
│  开始编写你的文章内容...                │ ← 编辑区
│                                         │
│                                         │
└─────────────────────────────────────────┘
```

点击工具栏按钮可以格式化文字，点击编辑区可以输入内容。
